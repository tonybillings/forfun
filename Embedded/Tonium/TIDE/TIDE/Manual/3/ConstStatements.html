xT50 :: Const Statements
The xT50 assembly language supports the use of named constant numbers and named registers. A const statement consists of the 'const' keyword, the name of the constant, an assignment operator, and the const value, each separated by one or more spaces. The name may contain letters, numbers and underscores, in any combination. The const keyword is not case-sensitive but the name and value are.

The following examples of const statements are all valid:

CONST led_bank = 225
Const On = 1
const OFF = 0
const INST_PTR = R0

Named constants may be used in place of instruction arguments only. The following two instruction statements (after the const statements) are equivalent:

const BTN_UP = 0
const BTN_DOWN = 1
const BTN_PRESSED = 2
const BTN_STATE = R10

SET R10 2
SET BTN_STATE BTN_PRESSED

Named constants can only be used in the scope that they are defined in and in any nested scopes within that scope. That is, constants defined in a core scope can only be used within that core; in a group scope then in all the cores within that group; at root scope then everywhere in the design. Named constants must be unique within the scope they are defined and all parent scopes of that scope.

In this example, a const statement is written at each of the three scopes:

const ROOT = 10

Group0.*::
const GROUP_0 = 20

Group0.Core0::
const CORE_0 = 30

SET R1 ROOT
SET R2 GROUP_0
SET R3 CORE_0

Group0.Core1::
const CORE_1 = 40

SET R1 ROOT
SET R2 GROUP_0
SET R3 CORE_1

Core2::
const CORE_2 = 50

SET R1 ROOT
SET R2 CORE_2

For more information on scopes, see the documentation regarding scope statements.
