xT50 :: Commenting Code
Single-line comments are supported with the use of the pound (#) character. Text following this character, on the same line as where this character is used, will be ignored in the compilation process.

The use of named constants and comments is highly recommended for clear and descriptive code. The following example illustrates this practice in good form:

# This program will blink an LED bank,
# consisting of 8 LED's, simultaneously.
# The blink rate can be adjusted by
# changing the clock rate or the
# delay instructions.
[CLOCK = 120]# Clock rate set to 120 MHz
const ON = 255# Turn on LED bank
const OFF = 0# Turn off LED bank
const rLED = r1# Register for current LED bank state
const mLED = 225# Memory address for LED bank
init:
set rLED OFF# Set initial state to off
begin_delay:
set r10 0# Reset first delay counter
set r11 0# Reset second delay counter
set r12 0# Reset third delay counter
delay:
add r10 r10 1# Increment first delay counter
beq r10 255# If first counter equals 255, skip to next counter
goto delay# Go back to first counter
add r11 r11 1# Increment second delay counter
beq r11 255# If second counter equals 255, skip to next counter
goto delay# Go back to first counter
add r12 r12 1# Increment third delay counter
beq r12 255# If third counter equals 255, toggle LED state
goto delay# Go back to first counter
toggle_state:
beq rLED OFF# If LED's are off, proceed to turn them on
goto turn_led_off# Proceed to turn LED's off
goto turn_led_on# Proceed to turn LED's on
turn_led_on:
set rLED ON# Set LED bank state to on
goto blink# Proceed to update LED bank
turn_led_off:
set rLED OFF# Set LED bank state to off
goto blink# Proceed to update LED bank
blink:
out rLED mLED # Update LED bank with new state
goto begin_delay# Restart delay loop
