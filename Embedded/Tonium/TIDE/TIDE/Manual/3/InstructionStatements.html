xT50 :: Instruction Statements
An instruction statement specifies the operation and zero or more arguments. The complete list of operations supported can be found here. The operation name and register name are not case-sensitive, however user-defined named constants and labels are. The argument list may be separated by commas and/or spaces. If the instruction contains a const argument it is not necessary to explicitly call the const version of the operation.

The following examples are all valid:

SETC R1, 123
SETC R1 123
setc r1 123
set r1 123
set r1 r2

The following example would not be valid since a register is specified where a const value is expected:

setc r1 r2

The only reason to explicitly call the const version of an operation is for compile-time validation of the argument list.
